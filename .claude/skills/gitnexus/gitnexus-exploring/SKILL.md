# Exploring Codebases with GitNexus

> Examples in this file use generic symbol names. GitNexus is language-agnostic — the workflow applies identically whether the indexed code is TypeScript, Python, Rust, Go, Java, C#, Ruby, PHP, or anything else GitNexus supports.

## When to Use

- "How does the trend scoring work?"
- "What's the project structure?"
- "Show me the main components"
- "Where is the favorites caching logic?"
- Understanding code you haven't seen before

## Workflow

```
1. READ gitnexus://repos                          → Discover indexed repos
2. READ gitnexus://repo/{name}/context             → Codebase overview, check staleness
3. gitnexus_query({query: "<what you want to understand>"})  → Find related execution flows
4. gitnexus_context({name: "<symbol>"})            → Deep dive on specific symbol
5. READ gitnexus://repo/{name}/process/{name}      → Trace full execution flow
```

> If step 2 says "Index is stale" → run `npx gitnexus analyze` in terminal.
> If `query` returns empty for a known repo → run `npx gitnexus index .` to register.

## Checklist

```
- [ ] READ gitnexus://repo/{name}/context
- [ ] gitnexus_query for the concept you want to understand
- [ ] Review returned processes (execution flows)
- [ ] gitnexus_context on key symbols for callers/callees
- [ ] READ process resource for full execution traces
- [ ] Read source files for implementation details
```

## Tools

**gitnexus_query** — find execution flows related to a concept:

```
gitnexus_query({query: "favorites caching"})
→ Processes: FavoritesRefreshFlow, DashboardLoadFlow
→ Symbols grouped by flow with file locations
```

**gitnexus_context** — 360-degree view of a symbol:

```
gitnexus_context({name: "trendAnalysisService"})
→ Incoming calls: VideoCard, AnalyserPage
→ Outgoing calls: calculateVelocityScore, calculateEngagementScore
→ Processes: AnalyserSearchFlow (step 4/6)
```

## Example: "How does trend scoring work?"

```
1. READ gitnexus://repo/tubetrend/context       → check freshness
2. gitnexus_query({query: "trend scoring"})
   → AnalyserSearchFlow: searchVideos → fetchVideoDetails → calculateTrendScore
3. gitnexus_context({name: "calculateTrendScore"})
   → Incoming: VideoCard, AnalyserPage
   → Outgoing: velocityScore, engagementScore
4. Read src/features/videos/services/trendAnalysisService.ts for implementation
```
