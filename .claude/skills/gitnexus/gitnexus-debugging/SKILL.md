# Debugging with GitNexus

## When to Use

- "Why is this function failing?"
- "Trace where this error comes from"
- "Who calls this method?"
- "Why is quota tracking off?"
- Investigating bugs, errors, or unexpected behavior

## Workflow

```
1. gitnexus_query({query: "<error or symptom>"})            → Find related execution flows
2. gitnexus_context({name: "<suspect>"})                    → See callers/callees/processes
3. READ gitnexus://repo/{name}/process/{name}                → Trace execution flow
4. gitnexus_cypher({query: "MATCH path..."})                 → Custom traces if needed
```

> If "Index is stale" → run `npx gitnexus analyze` in terminal.

## Checklist

```
- [ ] Understand the symptom (error message, unexpected behavior)
- [ ] gitnexus_query for error text or related code
- [ ] Identify the suspect function from returned processes
- [ ] gitnexus_context to see callers and callees
- [ ] Trace execution flow via process resource if applicable
- [ ] gitnexus_cypher for custom call chain traces if needed
- [ ] Read source files to confirm root cause
```

## Debugging Patterns

| Symptom              | GitNexus Approach                                          |
| -------------------- | ---------------------------------------------------------- |
| Error message        | `gitnexus_query` for error text → `context` on throw sites |
| Wrong return value   | `context` on the function → trace callees for data flow    |
| Intermittent failure | `context` → look for external calls, async deps            |
| Performance issue    | `context` → find symbols with many callers (hot paths)     |
| Recent regression    | `detect_changes` to see what your changes affect           |

## Tools

**gitnexus_query** — find code related to error:

```
gitnexus_query({query: "youtube api quota exhausted"})
→ Processes: AnalyserSearchFlow, ErrorHandling
→ Symbols: youtubeApiClient, YouTubeApiError, quotaService
```

**gitnexus_context** — full context for a suspect:

```
gitnexus_context({name: "youtubeApiClient.fetch"})
→ Incoming calls: searchService, channelService
→ Outgoing calls: fetch (browser), trackQuota
→ Processes: AnalyserSearchFlow (step 2/6)
```

**gitnexus_cypher** — custom call chain traces:

```cypher
MATCH path = (a)-[:CodeRelation {type: 'CALLS'}*1..2]->(b:Function {name: "trackQuota"})
RETURN [n IN nodes(path) | n.name] AS chain
```

## Example: "Search fails intermittently with 403"

```
1. gitnexus_query({query: "search error 403"})
   → Processes: AnalyserSearchFlow, ErrorHandling
   → Symbols: youtubeApiClient.fetch, YouTubeApiError

2. gitnexus_context({name: "youtubeApiClient.fetch"})
   → Outgoing: trackQuota, fetch (external)
   → Throws: YouTubeApiError (HTTP 403 → quota exhausted)

3. READ gitnexus://repo/tubetrend/process/AnalyserSearchFlow
   → Step 2 hits quota; intermittent 403 = daily quota near limit

4. Root cause: quota threshold not surfaced to UI before search submit
```
